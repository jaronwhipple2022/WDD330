MDN: Validating Forms

Client side validation is the pre check the web browser does before submitting the form data.

Client side is nly the PRECHECK. server side validation must be in place as well.

"Built-in form validation has better performance than JavaScript, but it is not as customizable as JavaScript validation."

Key words in html include: required, min, max, pattern, minlength, manxlength

Errors returned include: badInput, patternMismatch, rangeOverflow or rangeUnderflow, stepMismatch, tooLong or tooShort, typeMismatch, valueMissing, or a customError.

MDN: Using Fetch

Intro:
"The Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline,
such as requests and responses. It also provides a global fetch() method that provides an easy, logical way to fetch
resources asynchronously across the network."

basic fetch example:
fetch('http://example.com/movies.json')
.then(response => response.json())
.then(data => console.log(data));
